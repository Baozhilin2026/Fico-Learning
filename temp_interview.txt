
一、General SAP FICO Interview Questions  通用SAP FICO面试题

1. What is SAP FICO, and how does it integrate with other SAP modules?

中文：什么是SAP FICO？它如何与SAP其他模块集成？

Answer 英文：SAP FICO stands for Financial Accounting (FI) and Controlling
(CO). It's a core module in SAP ERP that manages financial transactions and
internal reporting.

FI (Financial Accounting): Deals with external reporting like balance
sheets, P&L statements, tax, etc.

CO (Controlling): Focuses on internal cost tracking, budgeting, and
profitability analysis.

Integration Examples:

MM (Materials Management): Integration through GR/IR accounts when goods
are received or invoiced.

SD (Sales & Distribution): Revenue postings when invoices are raised.

PP (Production Planning): Cost tracking of production processes.

HCM (Human Capital Management): Payroll results are posted to FI.

Answer 中文：SAP FICO是财务会计（FI）和管理会计（CO）的统称，是SAP
ERP的核心模块，用于管理财务交易与内部报表编制。

财务会计（FI）：负责资产负债表、利润表、税务等外部财务报告。

管理会计（CO）：聚焦企业内部成本跟踪、预算编制和盈利能力分析。

集成示例：

物料管理模块（MM）：收货或收票时，通过GR/IR清账科目实现集成；

销售与分销模块（SD）：开具发票时自动进行收入过账；

生产计划模块（PP）：对生产流程进行成本跟踪；

人力资本管理模块（HCM）：薪资核算结果过账至财务会计模块。

3. What is a Chart of Accounts (COA) in SAP FI?

中文：SAP FI中，会计科目表（COA）指的是什么？

Answer 英文：A Chart of Accounts (COA) is a list of all general ledger
accounts used by one or more company codes. It provides the structure for
categorizing financial transactions.

Types of COA:

Operating COA: Used for daily postings.

Group COA: Used for consolidation.

Country-specific COA: Used to meet local legal reporting needs.

Each G/L account in the COA contains details like account type, tax
category, etc.

Answer
中文：会计科目表（COA）是一个或多个公司代码所使用的全部总分类账科目的清单，为
财务交易的分类核算提供框架。

会计科目表类型：

运营会计科目表：用于日常业务过账；

集团会计科目表：用于集团合并报表编制；

国家特定会计科目表：满足当地法定报告要求。

会计科目表中的每个总分类账科目均包含科目类型、税务类别等详细信息。

5. What is the purpose of a fiscal year variant in SAP?

中文：SAP中会计年度变式的用途是什么？

Answer 英文：A Fiscal Year Variant defines the accounting year in terms of
periods and their start/end dates.

Types:

Calendar Year: Jan to Dec (e.g., 12 periods = 12 months).

Non-calendar Year: Can start in any month (e.g., Apr to Mar).

Can include special periods (up to 4) for year-end adjustments.

Used to match an organization's financial reporting cycle.

Answer 中文：会计年度变式用于定义会计年度的期间划分及各期间的起止日期。

类型：

日历年：1月至12月（如12个期间对应12个自然月）；

非日历年：可从任意月份开始（如4月至次年3月）。

会计年度变式可包含最多4个特殊期间，用于年末调整，其设置可匹配企业的财务报告周
期。

6. What are posting keys in SAP FI?

中文：SAP FI中的过账码指的是什么？

Answer 英文：A Posting Key is a two-digit numeric code that determines:

Account type (Customer, Vendor, G/L)

Debit or Credit posting

Field status (which fields are required/optional during entry)

Example:

40 - Debit entry for G/L

50 - Credit entry for G/L

01 - Customer Invoice

They streamline and control the posting process in journal entries.

Answer 中文：过账码是两位数字编码，用于确定以下内容：

科目类型（客户、供应商、总分类账）；

借贷方向；

字段状态（录入时哪些字段为必录/可选）。

示例：

40 - 总分类账借方过账；

50 - 总分类账贷方过账；

01 - 客户发票。

过账码可简化并管控记账凭证的过账流程。

7. What are field status groups, and how are they used in SAP FI?

中文：SAP FI中的字段状态组是什么？如何使用？

Answer 英文：Field Status Groups control the appearance
(optional/required/suppressed) of fields during G/L account posting.

Assigned in:

G/L account master record (in company code section)

For example:

For a cash account, the bank details field may be required.

For a revenue account, the cost center field may be optional.

Helps ensure data accuracy and relevance during transaction posting.

Answer
中文：字段状态组用于管控总分类账科目过账时各字段的显示状态（可选/必录/隐藏）
。

分配位置：

总分类账科目主数据的公司代码层。

示例：

现金科目中，银行明细字段设为必录；

收入科目中，成本中心字段设为可选。

字段状态组可确保交易过账时数据的准确性和相关性。

8. What is the purpose of document types in SAP FI?

中文：SAP FI中凭证类型的用途是什么？

Answer 英文：A Document Type determines:

The type of transaction (invoice, payment, etc.)

The number range used

The account types allowed

Examples:

SA - General Ledger

KR - Vendor Invoice

DR - Customer Invoice

This helps categorize and control financial transactions.

Answer 中文：凭证类型用于确定：

交易类型（发票、付款等）；

使用的编号范围；

允许过账的科目类型。

示例：

SA - 总分类账凭证；

KR - 供应商发票；

DR - 客户发票。

凭证类型可对财务交易进行分类和管控。

9. Explain the concept of parallel currencies in SAP FI？

中文：阐述SAP FI中平行货币的概念。

Answer 英文：Parallel currencies allow a company code to record
transactions in more than one currency.

Currency Types:

Local currency: Based on company code country

Global currency: For group reporting (usually USD or EUR)

Hard/Index currency: Used in certain countries for inflation accounting

This enables multi-national companies to comply with both local and group
reporting requirements.

Answer 中文：平行货币允许一个公司代码以多种货币记录交易。

货币类型：

本位币：根据公司代码所属国家确定；

集团货币：用于集团报告（通常为美元或欧元）；

硬货币/指数货币：部分国家用于通货膨胀会计核算。

平行货币可满足跨国企业的本地和集团双重报告要求。

10. What is a special G/L transaction in SAP FI?

中文：SAP FI中的特殊总分类账交易指的是什么？

Answer 英文：Special G/L Transactions are used for postings that are not
standard like:

Down payments

Bills of exchange

Guarantees

These postings are managed separately from normal items but are still tied
to the customer/vendor accounts.

They have:

Special G/L indicator

Separate reconciliation accounts

Example: Posting a down payment for a vendor before receiving
goods/services.

Answer 中文：特殊总分类账交易用于核算非标准的过账业务，例如：

预付款、汇票、担保。

此类过账与常规项目分开管理，但仍关联至对应的客户/供应商科目，其特征为：

带有特殊总分类账标识；

拥有独立的统驭科目。

示例：向供应商预付货款，尚未收到货物/服务时的过账。

二、 Automatic Payment Program (APP) Interview Questions
自动付款程序（APP）面试题

11. What is the Automatic Payment Program (APP) in SAP FI?

中文：SAP FI中的自动付款程序（APP）是什么？

Answer 英文：The Automatic Payment Program (APP) is a tool in SAP FI used
to automate vendor/customer invoice payments. It selects open invoices,
processes payments, and generates payment documents and files for bank
transfers or cheques.

Transaction Code: F110

Benefits:

Saves time

Minimizes errors

Supports mass payments

Automates selection, scheduling, and execution of payments

Answer 中文：自动付款程序（APP）是SAP
FI中的工具，用于实现供应商/客户发票付款的自动化，可筛选未清发票、处理付款业务
，并生成银行转账或支票所需的付款凭证及文件。

事务代码：F110

优势：

节省时间、减少错误、支持批量付款、实现付款的筛选、计划和执行自动化。

12. What are the main steps in configuring APP?

中文：配置自动付款程序（APP）的主要步骤有哪些？

Answer 英文：APP configuration is done via T-code: FBZP and includes:

All company codes: Define paying company codes

Paying company codes: Define payment parameters

Payment methods in country: Set up methods like cheque, transfer, etc.

Payment methods in company code: Assign payment methods at company code
level

Bank determination: Define ranking order, bank accounts, and charges

House banks: Define bank accounts used for payments

Forms: Define print program/form for checks or payment files

Answer 中文：自动付款程序通过事务代码FBZP配置，步骤包括：

所有公司代码：定义付款公司代码；

付款公司代码：定义付款参数；

国家层面付款方式：配置支票、转账等付款方式；

公司代码层面付款方式：在公司代码层分配付款方式；

银行确定：定义银行优先级、银行账户及手续费规则；

内部银行：定义用于付款的银行账户；

表单：定义支票或付款文件的打印程序/表单。

✅ 系统配置（FBZP）已完成。

15. What is the difference between a payment run and a proposal run in APP?

中文：自动付款程序（APP）中，付款运行与提议运行的区别是什么？

Answer

Proposal Run（提议运行）

用途：Simulates the payment without posting（模拟付款，不产生过账）

是否可冲销：Yes, can be edited or deleted（是，可编辑/删除）

使用场景：Review selected documents（核对筛选出的凭证）

Payment Run（付款运行）

用途：Executes the actual payment（执行实际付款）

是否可冲销：No, creates financial postings（否，会生成财务过账）

使用场景：Final step after proposal review（提议运行核对后的最终步骤）

16. How does bank determination work in APP?

中文：自动付款程序（APP）中，银行确定的工作原理是什么？

Answer 英文：Bank determination decides which bank account will be used for
a payment.

Configuration includes:

Ranking Order: Priority of banks for payment

Bank Accounts: House banks linked to G/L accounts

Available Amount: Minimum balance needed

Charges: Defines who pays transaction fees (payer/payee)

Set via: FBZP > Bank determination

Answer 中文：银行确定用于判定付款所使用的银行账户，其配置内容包括：

优先级：付款银行的先后顺序；

银行账户：关联总分类账科目的内部银行；

可用金额：账户所需的最低余额；

手续费：定义交易手续费的承担方（付款方/收款方）。

配置路径：FBZP>银行确定。

17. What are common errors encountered in APP, and how do you troubleshoot
them?

No payment method assigned（未分配付款方式）/ Assign payment method in

payment method settings（检查FBZP中的付款方式配置）

House bank not found（未找到内部银行）/Check bank determination and master
data（检查银行确定配置及主数据）

20. How do you configure payment terms in APP?

三、 Electronic Bank Statement (EBS) Interview Questions
电子银行对账单（EBS）面试题

21. What is the Electronic Bank Statement (EBS) in SAP FI?

中文：SAP FI中的电子银行对账单（EBS）是什么？

Answer 英文：EBS is an automated tool in SAP that allows banks to send
daily or periodic bank statements in electronic format (e.g., MT940, BAI,
CAMT). These statements are imported into SAP and automatically reconciled
with internal bank ledger entries, minimizing manual effort.

Answer
中文：电子银行对账单（EBS）是SAP中的自动化工具，支持银行以电子格式（如MT940、
BAI、CAMT）发送每日或定期银行对账单，该对账单可导入SAP系统，并与企业内部银行
总账记录自动对账，大幅减少人工操作。

22. How is EBS configured in SAP?

中文：如何在SAP中配置电子银行对账单（EBS）？

Answer 英文：EBS configuration involves several key steps:

Define House Bank and Account ID (T-code: FI12)

Assign Bank Statement Format (e.g., MT940, BAI, CAMT)

Define Transaction Types (e.g., debit/credit indicators)

Assign Posting Rules via T-code: OT83

Define Account Symbols (OT59) to map to G/L accounts

Create Interpretation Algorithms (for identifying postings)

Set Posting Parameters in transaction OBPM1 or FF_5

Answer 中文：电子银行对账单的配置包含以下关键步骤：

   1. 定义内部银行和账户ID（事务代码FI12）；

   2. 分配银行对账单格式（如MT940、BAI、CAMT）；

   3. 定义交易类型（如借方/贷方标识）；

   4. 分配过账规则（事务代码OT83）；

   5. 定义科目符号（OT59），并映射至总分类账科目；

   6. 创建解析算法（用于识别过账对象）；

   7. 在事务代码OBPM1或FF_5中设置过账参数。

24. What is the difference between EBS and Manual Bank Statement (MBS)?

中文：电子银行对账单（EBS）与手工银行对账单（MBS）的区别是什么？

Answer

EBS (Electronic Bank Statement)

录入来源：File from bank (e.g., MT940, BAI)（银行提供的电子文件）

自动化程度：Fully automated（完全自动化）

过账与对账：Automatic using posting rules（通过过账规则自动过账、对账）

效率：High - Suitable for large volumes（效率高，适用于大交易量）

MBS (Manual Bank Statement)

录入来源：Manually entered by user（用户手工录入）

自动化程度：Manual data entry（手工录入数据）

过账与对账：Manual matching and posting（手工匹配并过账）

效率：Low - Time-consuming（效率低，耗时）

25. What is the role of interpretation algorithms in EBS?

中文：电子银行对账单（EBS）中，解析算法的作用是什么？

Answer 英文：Interpretation algorithms help SAP interpret which document or
account should be posted for each bank statement line.

Examples:

001 - Posting based on document number in note to payee

015 - Search for reference in text field

019 - Post using bank account number and customer/vendor

Configured in transaction OT83.

Answer
中文：解析算法用于帮助SAP系统识别银行对账单的每个行项目应过账至哪个凭证或科目
。

示例：

001 - 根据付款附言中的凭证编号过账；

015 - 在文本字段中搜索参考信息进行匹配；

019 - 通过银行账号和客户/供应商信息进行过账。

解析算法在事务代码OT83中配置。

26. How do you define bank statement posting rules in SAP?
