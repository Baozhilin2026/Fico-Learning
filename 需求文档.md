SAP FICO模块中文学习英语网站开发需求文档
1. 文档说明
1.1 文档目的

本文档详细描述SAP FICO模块中文学习英语网站的全部开发需求，明确网站功能、界面、技术、数据等各项要求，为开发人员（Claude Code）提供清晰、可落地的开发依据，确保开发成果与预期一致，无需额外反复确认需求细节。

1.2 适用对象

开发人员（Claude Code）、需求提出方（我方），用于开发对接、需求校验及验收。

1.3 核心前提

1.  网站核心定位：SAP FICO模块专属英语学习平台，以“中文辅助+英语应用”为核心，聚焦FICO相关英语的“认、听、读、写、用”，避免枯燥化，适配SAP FICO顾问的学习、工作及面试场景；

2.  核心数据提供：单词和短语、项目沟通常用语句、英文面试内容，均由我方提供JSON格式数据，开发方负责解析、展示及适配相关功能；

3.  授权验证要求：用户打开网站后，需输入我方提供的授权码，验证通过方可使用全部功能，未授权仅可查看网站简介（无核心功能访问权限）。

2. 总体需求

2.1 网站定位

专为SAP FICO顾问、备考FICO实施顾问岗位者、需提升FICO相关英语沟通/面试能力的从业者设计，打造“场景化、互动化、实用化”的FICO英语学习平台，核心目标：让用户看到FICO英文术语能认识，听到能默写，熟练运用到工作沟通、面试中。

2.2 核心功能模块

网站共包含5大核心模块，按用户使用流程及重要性排序：授权验证模块、单词和短语学习模块、项目沟通常用语句模块、英文面试内容模块、AI模拟面试模块，各模块独立存在且可快速切换，界面风格统一、操作便捷。

2.3 整体交互要求

1.  操作简洁：核心功能入口清晰，每步操作不超过3次点击，避免复杂跳转；

2.  趣味性：避免枯燥的纯文字展示，融入互动练习、进度反馈、场景化呈现等设计，提升学习积极性；

3.  朗读适配：沟通语句、面试内容支持多口音（印度、新加坡、欧美）、男女朗读者切换，朗读清晰可重复播放，支持暂停、倍速（0.8x、1x、1.2x）调节；

4.  响应式：适配电脑端（核心使用场景）、平板端，确保不同设备打开后界面布局合理、功能正常使用；

5.  数据兼容：支持我方后续更新JSON格式的单词、语句、面试内容，无需修改代码，仅需上传JSON文件即可完成数据更新。

2.4 授权验证核心要求

1.  启动流程：用户打开网站 → 进入授权验证页面（无其他入口） → 输入授权码 → 验证通过 → 进入网站首页；验证失败 → 提示“授权码错误，请重新输入”，无法进入核心功能；

2.  授权码规则：采用随机字符串方案，由我方负责生成，具体规则：长度为8-16位，由大小写英文字母（A-Z、a-z）+ 数字（0-9）组成，可按需添加特殊符号（如!、@、#），无固定规律，确保唯一性和安全性；开发方无需关注随机字符串的生成逻辑，仅负责“输入校验+验证通过后解锁功能”，验证逻辑：简单匹配校验（我方提供校验规则，或开发方预留接口，我方后续配置，仅校验输入字符串与我方生成的有效随机字符串是否一致）；

3.  未授权提示：未输入授权码/授权失败时，仅显示网站名称、简介（核心功能简要说明）、授权码输入框、“忘记授权码？联系管理员”提示（管理员联系方式由我方提供，可修改），无任何核心功能入口。

3.2.3 词条展示格式（每词条固定7项内容，我方JSON提供对应字段）

每个单词/短语词条以“卡片式”展示，卡片包含以下7项内容，布局清晰、重点突出，支持点击展开/收起（默认展开核心内容：英文术语、IPA、中文翻译、中文解释；点击后展开剩余内容：例句、会议说法、错误用法）：

1.  英文术语：加粗显示，字体大于其他内容，作为词条核心标识；

2.  英语读音（IPA）：紧跟英文术语下方，标注国际音标，支持点击“播放”按钮朗读（默认英文原声，男女声可切换，无需多口音）；

3.  中文翻译（标准叫法）：加粗显示，对应SAP FICO行业标准中文术语，避免直译；

4.  中文解释（业务语义）：详细说明该术语在SAP FICO业务中的含义、应用场景，不是英文直译，贴合顾问实际工作理解；

5.  SAP 场景例句：英文例句（贴合SAP操作/业务场景），紧跟中文翻译（例句对应翻译），支持点击朗读例句；

6.  常见会议说法：该术语在FICO项目会议中常用的英文表达（短句形式），支持点击朗读；

7.  常见错误用法：明确标注该术语易混淆、易误用的英文表达，搭配正确用法对比，强化记忆。

3.2.4 互动学习功能（核心，避免枯燥）

围绕“看到能认识、听到能默写”设计，每个场景下均可使用以下功能：

1.  听力默写练习：

   - 点击“听力默写”按钮，系统随机播放当前场景下的单词/短语读音（IPA对应的读音，可切换男女声）；

   - 用户在输入框中默写对应的英文术语，提交后系统自动校验（忽略大小写，精准匹配拼写）；

   - 校验结果反馈：正确则提示“默写正确！”，显示该词条核心信息（英文、中文翻译）；错误则提示“默写错误，正确答案：XXX”，并播放正确读音，可重复播放、重新默写；

   - 支持“连续默写”（自动播放下一个读音）、“重新播放”（重复当前读音）、“跳过”（跳过当前单词/短语）。

2.  单词认读练习：

   - 系统随机展示当前场景下的中文翻译/中文解释，用户选择对应的英文术语（多选题/单选题，可配置）；

   - 提交后显示正确答案，搭配读音播放，强化“中文→英文”的关联记忆。

3.  收藏功能：

   - 每个词条卡片右上角有“收藏”按钮，点击后收藏该词条，收藏的词条集中在“我的收藏”页面（可单独查看、取消收藏）；

   - 收藏的词条可单独进行听力默写、认读练习，方便用户重点攻克难点术语。

4.  进度记录：

   - 记录用户当前场景下的学习进度（已学习词条数/总词条数）、听力默写正确率、认读练习正确率；

   - 显示“今日学习时长”“累计学习词条数”，以进度条、小图标形式展示，提升学习成就感。

3.2.5 数据适配

1.  数据加载：开发方负责解析我方提供的JSON格式单词/短语数据，JSON字段需包含：场景分类、英文术语、IPA、中文翻译、中文解释、SAP场景例句（英文+中文）、常见会议说法、常见错误用法（错误+正确），字段名可由双方协商确定；

2.  数据更新：支持我方后续上传新的JSON文件，替换原有数据，无需修改代码，上传后系统自动解析、更新所有词条及场景；

3.  异常处理：若JSON数据缺失某字段（如某词条无常见错误用法），则该字段不显示，不影响整个词条及模块正常使用。

3.  朗读者切换：支持男女声切换，下拉菜单选择，与口音组合使用（例：印度口音-女声、欧美口音-男声），统一应用于所有语句；

4.  播放控制：每个语句（列表项/详情页）均有“播放”按钮，支持暂停、重复播放、倍速调节（0.8x、1x、1.2x），播放时显示播放进度条；

5.  跟读提示：播放结束后，可提示“请跟读”，用户可点击“重新播放”反复模仿，提升口语表达能力。

每个语句（列表项/详情页）均有“播放”按钮，支持以下配置，操作便捷：

4.  跟读提示：播放结束后，可提示“请跟读”，用户可点击“重新播放”反复模仿，提升口语表达能力。

3.5 AI模拟面试模块（核心模块4）

3.5.1 模块核心目标

模拟SAP FICO实施顾问岗位的真实英文面试场景，AI作为面试官，用户作为应聘者，完成面试流程后，AI对用户的回答进行专业点评（聚焦FICO业务专业性、英语表达准确性），帮助用户提升面试能力。

3.5.2 面试前提

1.  面试岗位：固定为“SAP FICO实施顾问”，无需用户选择；

2.  面试语言：全程英文（AI提问为英文，用户回答需输入英文，支持中文辅助输入后自动翻译为英文，可关闭该功能）；

3.  面试口音：AI提问支持印度、新加坡、欧美3种口音，男女声切换，用户可在面试开始前选择，面试过程中不可切换。

模拟面试流程（固定流程，不可修改，贴合真实面试）

1.  面试准备：用户进入模块后，选择AI面试官的口音、男女声，点击“开始面试”，系统提示“面试即将开始，全程英文提问，请用英文回答，可点击录音回答（可选）或输入回答”；

2.  面试环节（按以下顺序自动推进，AI提问从我方提供的JSON面试内容中随机抽取，每个环节提问数量可配置，默认各2-3题）：

   - 第一环节：自我介绍（AI提示“Please introduce yourself briefly”，用户输入/录音回答）；

   - 第二环节：项目经验提问（AI提问与FICO项目实施相关的英文问题，如项目中负责的FICO模块、遇到的问题及解决方法等）；

   - 第三环节：技术问题提问（AI提问FICO模块相关的技术类英文问题，如FI-AA资产核算流程、CO成本中心配置等）；

   - 第四环节：场景题提问（AI提问FICO项目中的真实场景问题，如与财务总监沟通配置需求的话术、解决Keyuser提出的业务疑问等）；

   - 第五环节：反问环节（AI提示“Do you have any questions to ask?”，用户可输入/录音英文反问，也可选择“无问题”）；

3.  回答方式（用户二选一）：

   - 输入回答：在输入框中输入英文回答，支持中文输入后自动翻译为英文（可关闭翻译功能），输入完成后点击“提交回答”；

   - 录音回答：点击“录音”按钮，录制英文口头回答，录音时长限制（默认1-3分钟/题，可配置），支持暂停、重新录音，录制完成后点击“提交录音”（录音仅用于当前面试点评，不长期存储）；

4.  面试结束：反问环节结束后，系统提示“Interview ended, please wait for the comment”，AI在3-5秒内生成点评报告。

3.5.3 面试点评功能（核心）

点评报告聚焦“用户回答部分”，分为3个维度，内容具体、有针对性，不空洞，贴合SAP FICO实施顾问岗位要求：

1.  英语表达点评：

   - 优点：如“发音清晰（录音回答）、语法正确、语句流畅，符合面试沟通场景”；

   - 不足：如“存在语法错误（例：XXX）、发音不准确（例：XXX术语读音错误）、语句过于简单，缺乏专业表达”；

   - 建议：如“加强FICO高频术语的发音练习，多使用项目沟通中的常用语句，提升语句流畅度”。

2.  FICO业务专业性点评：

   - 优点：如“对FI-AA资产会计流程的回答准确，贴合SAP实际配置场景，能清晰说明业务逻辑”；

   - 不足：如“对CO成本中心核算的理解有误，混淆了成本中心与内部核算的业务边界，未结合SAP场景说明”；

   - 建议：如“重点复习CO模块的核心业务逻辑，结合项目场景记忆业务术语及流程，提升专业应答能力”。

3.  面试技巧点评：

   - 优点：如“自我介绍简洁明了，重点突出FICO项目经验；反问环节的问题贴合岗位需求，体现对项目的关注度”；

   - 不足：如“回答问题过于冗长，未突出重点；遇到不会的问题未及时礼貌回应，影响面试体验”；

   - 建议：如“回答问题时先给出核心答案，再补充细节；遇到不会的问题可回应‘I need a little time to think about it’，避免沉默”。

4.  点评附加功能：

   - 支持查看“AI推荐参考答案”（对应每个提问的标准英文参考答案，我方JSON提供）；

   - 支持播放AI提问的读音、用户录音回答的读音，对比改进；

   - 支持将点评报告导出为PDF（可选功能，优先实现核心点评功能）。

3.5.4 辅助功能

1.  面试记录：保存用户最近3次的模拟面试记录（包含面试提问、用户回答、点评报告），可查看、删除，超过3次自动覆盖最早的记录；

2.  重新面试：面试结束后，用户可点击“重新面试”，重新选择口音、男女声，开始新的模拟面试；

3.  面试中断：面试过程中，用户可点击“中断面试”，系统提示“确定要中断面试吗？中断后本次面试记录不保存”，确认后返回模块首页。

3.5.5 数据适配

1.  数据加载：开发方解析我方提供的JSON格式英文面试内容数据，用于AI提问、参考答案生成，JSON字段需包含：面试环节、英文提问、中文翻译、英文参考答案、中文参考答案，字段名可协商确定；

2.  AI点评逻辑：开发方实现AI点评的基础框架，点评内容结合用户回答、我方提供的参考答案及FICO岗位核心要求，确保点评有针对性、专业性（无需开发复杂AI模型，基于关键词匹配+固定点评模板，结合用户回答内容动态生成）；

3.  异常处理：若用户未提交回答（超时，默认30秒/题，可配置），AI提示“Time is up, please proceed to the next question”，并在点评中注明“该题未回答，建议加强相关知识点的练习”。
